1. 前言

  1. 连接mysql端口3306(一般为了安全起见会将默认端口修改，修改后则需要连接修改后的端口)
  2. 连接到mysql服务（mysql数据库）的指令 mysql [-h 主机ip] [-p 端口号] -u 用户名 -p[密码]
    1. -p密码没有空格
    2. -p后面没有写密码回车会要求 输入
    3. -h -p可以不写默认本机和3306
  3. 所谓安装数据库就是在主机上安装了一个数据库管理系统DBMS，这个管理程序可以管理多个数据库
  4. 一个数据库中可以创建多个表，用来保存数据
  6. 数据库储存在哪
    1. 安装路径下data目录下
    2. 表 在数据库目录下
  7. mysql数据库-普通表的本质仍然是文件（更利于数据管理的文件格式）（有些是映射在内存中，有些是硬盘内存共用）
  8. 数据在数据库中的储存方式
    1. 主要是表（行 row 列 column）
    2. 表的一行被称为一条记录（在java程序中往往使用一个对象来表示一条记录）
  9. sql语句分类
    1. DDL：数据定义语句【create 表库】
    2. DML:数据操作语句【insert，update , delete】
    3. DQL:数据查询语句【select】
    4. DCL:数据控制语句【管理数据库：不如用户权限，grant， revoke】
  11. 启动mysql 服务 : net start mysql
  12. 停止mysql服务指令 : net stop mysql

4.3 数值型小数的使用

DECIMAL[M, D]

可以支持更加精确地小数位，M是整体长度总数，D是小数点（标度）后面的位数（例：123.45678 M = 8，D = 5）

如果D是0， 则值没有小数点或分数部分，M最大==65==，D最大==30==，D默认0， M默认10

建议：高精度小数推荐使用DCIMAL

5.4.3 group by子句

将待查询序列按照要求分组，对每个组进行过滤查找查找

当一个语句中同时含有where、group by 、having及聚集函数时，执行顺序如下：

  1. 执行where子句查找符合条件的数据；
  2. 使用group by 子句对数据进行分组；对group by 子句形成的组运行聚集函数计算每一组的值；
  3. 最后用having 子句去掉不符合条件的组。

需要注意的是，

  1. having 子句中的每一个元素也必须出现在select列表中。有些数据库例外，如oracle.
  2. having子句和where子句都可以用来设定限制条件以使查询结果满足一定的条件限制。
  3. having子句限制的是组，而不是行。where子句中不能使用聚集函数，而having子句中可以。

当加上其他sql语句时，执行顺序如下：

select –>where –> group by–> having–>order by

11. mysql约束

11.1 主键

唯一标识表中记录的列，主键列数据不能重复

复合主键写在末尾，priamry key (列名， 列名2) 列名一定要用括号包围，即使只有一个列作为主键，如果写在属性最后就不用括号

细节：

  1. 主键不能重复，且不能为空，相当于使用了unique not null
  2. 主键可以使用多个列同时作为主键（复合主键），多个列的组合不能重复即可

11.2 not null

如果在列上定义了not null 则表示插入数据时必须为列提供数据

11.3 unique

当定义了唯一约束后，该列值时不能重复的

如果字段没有定义not null ，则可以添加多个空，即使使用了unique标识

11.4 外键 foreign key

将主表中的主键约束列或者unique约束列和从表中的外键列建立联系，要求外键列数据必须在主键列存在或者为空

主表要求具有主键约束或者unique约束

语法

注意事项：

  1. 外键指向表的字段要求是primary key或者是unique
  2. 表引擎为innodb才支持外键
  3. 外键字段的类型要与主键字段的类型一致，长度可以不同
  4. 外键字段的值必须在主键字段中出现过，或者为空
  5. 一旦建立外键关系数据就不能随意删除，欲删除主键字段必须将外键中与该字段关联的外键数据全部删除方可删除主键字段

提高数据库性能，不用加内存，不用改程序，不用调sql，查询速度提高百倍千倍

  1. 索引本身会占用空间
  2. 创建索引后只会对创建缩印的列有效，其他列查询速度依旧

索引的原理：

  • 没有索引为什么会慢？
    • 因为每次查询都会进行全表查询
  • 使用索引为什么快？
    • 形成一个数据结构，比如二叉树，B树，B+树等

  • 对update，delete， insert语句的效率影响，因为会维护索引

    一般开发不使用mysql自带的全文索引，而是使用全文搜索Solr和ElasticSearch（Es）

14. mysql 事务

14.1 事务管理

事务用于保证数据的一致性，他由一组相关的dml语句组成，该组语句要不全部成功要不全部失败

例如：转账，如果转出方转出失败，接收方就不能接受成功，反之亦然

  • 事务和锁

    当执行事务操作时，mysql会在表上加锁，防止其他用户修改表的数据，在实务操作commit之前的所有操作其他用户是看不见的

  • mysql数据库控制台事务的重要操作

    • start transaction 开始一个事务
    • savepoint 保存点名字 设置保存点
    • rollback to 保存点名字 回退事务
    • rollback 回退全部事务
    • commit 提交事务，删除保存点，释放锁，所有的操作生效，不能回退

    注意事项：

    1. 回退以后，该保存点后面的保存点将被删除
    2. rollback回退至事务建立之初
    3. 如果不开启事务，是dml自动提交的，不能回滚
    4. 开启事务是默认创建一个保存点，即rollback回退的保存点
    5. innodb存储引擎才支持事务机制
    6. 开启事务也可以用set autocommit=off

14.2 事务隔离级别

脏读：当一个事务读取到了另一个事务未提交的修改

不可重复读：同一个查询在同一个事务中多次进行，由于其他提交事务所做的修改删除，每次返回不同的结果

幻读：同一个查询在同一个事务中多次进行，由于其他提交事务所做的插入，每次返回不同的结果

  • mysql的隔离级别有哪些？（√表示会出现，×表示不会出现）
隔离级别 脏读 不可重复读 幻读 加锁读
读未提交（read uncommitted） √ √ √ ×
读已提交（read committed） × √ √ ×
可重复读（repeatable read） × × × ×
可串行化（serializable） × × × √

  • 常用指令

    • 查看当前会话隔离等级

    • 查看系统当前隔离等级

    • 设置当前会话隔离等级

    • 设置当前系统隔离等级

  • mysql默认事务等级是repeatable read

16. 视图（view）

视图是一个虚拟表，其内容由查询定义，同真实的表一样，视图包含列，其数据来自对应的真实的表，通过视图是可以修改基表，修改基表也会影响视图。

视图的基本使用

  1. create view 视图名 as select 语句
  2. alter view 视图名 as select语句
  3. show create view 视图名
  4. drop view 视图名

细节：

  1. 当我们创建视图后去对应的数据库文件查看只能看到视图结构文件，没有数据文件，数据仍来自基表
  2. 视图的数据变化会影响到基表，基表变化也会影响视图（就像java引用，指向同一个数据文件，无论从哪边修改都会修改成功）
