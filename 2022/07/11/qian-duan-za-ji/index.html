css

层叠样式表，网页实际上是一个多层的结构，通过css可以对网页的每一层进行设置样式，而最终我们只能看到最上面的一层

使用css修改样式

1、在标签内部通过style属性设置元素的样式（内联样式，行内样式）

缺点：只能影响到一个元素，并且当样式发生变化的时候不方便维护

注意：开发者绝对不要使用内联样式！！！

2、内部样式表：将样式编写到head中的style标签里,然后通过css选择器来选中元素，并为其设置样式

可以同时设置多处样式，并且修改只需修改一处即可

缺点：只能对一个网页使用，里面的样式无法跨页面使用

3、外部样式表：将css样式编写到一个css文件中，然后通过link标签引入文件

开发中最推荐的方式。

优点：

只要想使用这个样式的文件都可以引用，使样式可以在不同的页面中复用

将样式写入外部的css文件中，可以充分使用浏览器的cache，提升用户体验

内边距

内容区和边框之间的距离

一共有四个方向的内边距

  1. padding-top

  2. padding-right

  3. padding-bottom

  4. padding-left

内边距的设置会影响到盒子的大小

背景颜色会延伸到内边距上

一个盒子的可见宽的大小有内容区，内边距，和边框共同决定，所以计算盒子大小的时候需要将这三个区域加到一起计算。

内边距指定方法和边框 border-width 相同。

外边距

margin 不会影响可见宽的大小，但是会影响盒子的位置

margin-top 上

margin-left 左

margin-bottom 下

margin-right 右

或者直接margin:100px 100px 100px 100px

margin 会影响到盒子实际占用空间的大小

水平布局

  • 一个元素的水平布局必须满足以下等式

  • 左外边距+左内边距+左边框+width+右内边距+右边框+右外边距=父元素的内容区的宽度（width）

  • 如果计算结果是等式不成立，则称过度约束，则等式会自动调整

  • 如果7个值中没有auto，则会自动调整margin-right的值

  • 如果有auto就会调整auto，width的值默认是auto

  • 如果将一个宽度和外边距同时设置为auto，那么宽度会调整到最大，设置为auto的外边距会设置为0

  • 如果将三个值都设置为auto，那么外边距都为0

垂直方向的布局

父元素的宽度 height 如果不写会被子元素撑开

如果父元素的height的长度小于子元素就会溢出

处理溢出的方法：

使用属性overflow

  1. visible：默认值，子元素会从父元素中溢出，在父元素外部的位置显示
  2. hidden：溢出部分会被裁去
  3. scroll：生成两个滚动条
  4. auto：根据需要生成

（垂直）外边距的重叠

相邻的 垂直方向的外边距会发生重叠，上面元素的下外边距和下面元素的上外边距

  1. 兄弟元素：两者之间取较大值做为外边距（两者都是正值，如果都是负值，去较小的）

  2. 父子元素：

    1. 父子元素间相邻外边距子元素的会传递给父元素（上外边距）

    2. 父子外边距的折叠会影响到页面的布局，必须要进行处理

    3. 处理办法

      • 不用外边距，使用内边距，上面加内边距，下面减内边距

      • 加边框(两个元素就不相邻了)，把两个外边距隔开

行内元素的盒子模型

  1. 行内元素 不支持 设置内容区宽高

  2. 行内元素可以设置padding，margin，border但是垂直方向不会影响元素布局

  3. 设置元素显示的类型 display

    1. inline：设置为行内元素
    2. block：将元素设置为块元素
    3. inline-block：行内块元素，既可以设置宽高，也不会独占一行
    4. none：隐藏元素（可以用来隐藏菜单，鼠标移到上面显示），不会占据空间

  4. 设置元素显示状态visibility

    1. visible：默认值，元素在页面后中显示
    2. hidden：元素在页面中隐藏，不显示，但是空间被占据了

圆角

浮动

通过浮动可以是一个元素向其父元素的左侧或右侧移动，设置浮动以后他就脱离文档流了，不在占用文档流的位置，所以元素下边的还有文档流中的元素会自动向上移动

使用float属性来设置元素的浮动

可选值：

  1. none：默认值，元素不浮动
  2. left：元素向左浮动
  3. right：元素向右移动

特点：

  1. 由于内容区，左右内外边距之和为定值的要求，所以元素的右边一般会将父元素填充慢外边距，导致下一个元素不能出现在这一个元素的水平方向
  2. 设置浮动后将解除这个限制，不再会自动填充，因为它已经向上一层，离开文档流了
  3. 设置浮动后，会向左侧或者右侧移动，但是浮动元素默认不会从父元素中移出
  4. 浮动元素在他那一层，其他元素也是无法盖住本层元素的。
  5. 如果浮动元素上面有一个非浮动元素，那么这个浮动元素还是没法移到上面的
  6. 浮动元素不会超过上面的浮动元素的兄弟，最多和他一样高

总结：浮动的作用就是让页面中的元素可以水平排列，通过浮动，可以制作一些水平布局

浮动元素不会盖住文字，文字会自动围绕在浮动元素 的周围，所以可以用浮动来设置文字环绕图片的效果

元素设置浮动后，会从文档流中脱离，元素的一些特点也会发生变化

脱离文档流的特点

块元素

  1. 块元素不再独占页面一行
  2. 块元素宽度和高度默认被内容撑开

行内元素

  1. 会变成块元素，特点和块元素一样，脱离文档流后，行内元素和块元素都一样了

BFC

用BFC (块级格式化布局)，css中的一个隐藏属性

特点：

  1. 开启后的元素不会被浮动元素覆盖
  2. 开启后子父元素的外边距不会重叠
  3. 开启后的元素可以包含浮动的子元素

开启方式：

  1. 给该元素设置浮动（会丢失宽度）
  2. 将元素设置为行内块元素(丢失宽度)
  3. 将元素的 overflow 设置为非 visible 值( hidden )

clear

如果我们不希望某个元素 因为其他元素的浮动的影响导致位置的改变

可以通过 clear 属性来清除浮动元素对当前元素产生的影响

可选值：

  1. left：清除左侧浮动元素对当前元素的影响
  2. right：清除右侧浮动元素对当前元素的影响
  3. both：清除两侧中影响最大的一侧

原理：

设置清除浮动后，浏览器会自动为元素添加上外边距，以使其位置不受其他元素的影响。

定位

三个盒子，如果想要让中间的盒子跑到第一个盒子旁边，其他盒子不动，怎么办？

传统方法：

  1. 设置盒子2的marginleft让他向左
  2. 设置margintop，让他向上
  3. 但是盒子3也上去了
  4. 设置3的上外边距将位置调回去

这样的方法，耦合太高，调一个元素的位置，需要调整好几个元素

position 一种更加高级的布局手段，通过定位可以把元素摆放到页面的任意一个位置

使用属性position

参数值：

  1. static 默认值，没有开启定位
  2. relative 开启元素的相对定位
  3. absolute 开启元素的绝对定位
  4. fixed 开启元素的固定定位
  5. sticky 开启元素的粘滞定位

相对定位

开启相对定位后如果不设置偏移量，没有任何变化

偏移量（offset）：

可以设置元素的位置

  • left
    • 定位元素和定位位置上边的距离，可以想象为外边距，但是不会影响别人，下面的属性以此类推
  • right
  • top
  • bottom

特点：

  • 相对定位会提升元素的层级
  • 相对定位不会使元素脱离文档流
  • 相对定位不会改变元素的性质（行内还是行内）

包含块：

  • 正常情况下是离当前元素最近的祖先块元素
  • 绝对定位的包含块：包含块就是离他最近的开启了定位的祖先元素
  • 如果所有的祖先元素都没有开启定位，则相对于根元素进行定位

绝对定位

特点：

  • 开启绝对定位后，如果不设置偏移量，元素的位置不会发生变化
  • 开启绝对定位后，元素会从文档流中脱离
  • 绝对定位会改变元素的性质（行内变成块， 块的宽高被内容撑开）
  • 绝对定位会使元素提升层级
  • 绝对定位元素元素是相对于其包含块进行定位

固定定位

也是一种绝对定位，大部分特点都要和绝对定位一样

唯一不同的是固定定位永远参考与浏览器的 视口 进行定位

固定定位的元素不会随着网页滚动条的滚动

粘滞定位

粘滞定位和相对定位的特点基本相同，不同的是可以在元素到达某个位置将其固定

绝对定位元素的位置

  • 当我们开启绝对定位后，水平方向的布局就要添加两个元素，left 和 right ，此时的规则和以前一样，只是多了两个值
  • 当发生过度约束的时候：
    • 如果没有auto 自动调整 right
    • 如果有auto则调整auto
  • 可以设置auto的值：
    • margin width left right
  • 因为 left 和 right 的默认值就是auto， 所以当等式不满足的时候就会自动调整auto的值，以使等式成立
  • 垂直方向的布局同样如此

层级

背景

background-color：背景颜色

background-image：背景图片

  • 可以同时设置背景图片和背景色，这样背景颜色会成为图片的背景色
  • 如果背景图片小于元素，则背景图片会自动平铺，将元素充满

background-repeat：设置背景重复方式

  • repeat：默认值，背景会沿着x轴方向和y轴方向重复
  • repeat-x：沿着x轴方向
  • repeat-y：沿着y轴方向
  • no-repeat：不重复

background-position：设置背景图片的位置

  • 通过top、left、right、bottom、center基本表示方位的词来设置位置（9宫格定位，至少需要两个词，如果只写一个则第二个默认center）
  • 通过偏移量指定位置，需要两个值，水平方向和垂直方向

background-clip：设置背景的范围：

  • border-box：默认值，背景会出现在边框的下边
  • padding-box：只出现在内容区和内边距，不会出现在边框下
  • content-box：只出现在内容区下

background-origin：背景图片的偏移量计算的原点

  • padding-box：默认值，从内边距出开始计算
  • conter-box：从内容区开始计算
  • border-box：背景图片的原点从边框出开始计算

background-size：背景图片的尺寸

  • 设置宽度和高度
  • 如果只写一个值，另一个默认 auto
  • 可选值
    • cover：图片比例不变，将元素充满
    • contain ：图片比例不变，将图片在元素中完整显示

background-attachment：设置元素背景图片是否跟随元素移动

  • scroll：默认值，移动
  • fixed：不移动

background：背景相关的简写属性

  • origin 和 clip 两个样式 orgin 必须在 clip 前面
  • size 必须写在 position 后面，并且使用 / 隔开

浏览器中的资源是按需加载

解决图片闪烁：

将多个小图片放到一个大图片中，这样图片会同时加载，通过调整图片的background-position 来调整显示图片的那部分，这个在网页中应用十分广泛，被称为(css - sprite,雪碧图)

雪碧图使用步骤：

  1. 确定使用的图标
  2. 测量图标的大小
  3. 根据测量结果来创建元素
  4. 将雪碧图设置为元素的背景图片
  5. 设置一个偏移量，以显示正确的图片

雪碧图特点：

  • 一次性将多个图片加载进页面，降低请求次数，加快访问速度，提升用户体验

    • 大小：
