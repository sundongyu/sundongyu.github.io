操作系统

1、前言

操作系统的特征

  • 并发（交替执行），并行是同时发生
  • 共享，资源共享，供内存中多个并发的程序使用
    • 互斥：例如摄像头，同一时间只能一个进程访问
    • 同时共享：例如磁盘，多个进程可同时访问，微观上是交替访问
  • 虚拟：把一个物理上的实体变为若干个逻辑上的对应物
    • 空分复用（虚拟存储技术）
    • 时分复用（虚拟处理器）
  • 异步：资源有限，即使支持并发，进程的执行不是一管到底的，是走走停停的。

没有并发和共享就谈不上虚拟和异步，因此并发和共享是操作系统的两个最基本的特征。

操作系统的发展和分类

  • 手工操作阶段
  • 批处理阶段–单道批处理系统（引入脱机输入输出），缓解了人机速度矛盾，资源利用率任然很低
  • 批处理阶段–多道批处理系统（多道程序并发执行，资源利用率高），不提供人机交互
  • 分时操作系统 ：计算机以时间片为单位轮流为用户 / 作业服务，各个用户可通过终端与计算机交互，解决了人机交互问题，不能处理紧急任务
  • 实时操作系统：及时性、可靠性，能优先处理紧急任务
    • 硬实时：绝对严格的时间之内处理
    • 软实时：能接受偶尔违反时间规定
  • 网络操作系统
  • 分布式操作系统

2、OS 的运行机制和体系结构

操作系统的运行机制

什么是指令？

处理器能识别、执行的最基本的命令

两种指令

  • 特权指令
  • 非特权指令

两种处理器状态

用程序状态寄存器（psw）中的某标志位来标识当前处于什么状态

  • 用户态（目态），此时只能执行非特权指令
  • 核心态（管态）

两种程序

  • 内核程序：系统的管理者，可以执行所有指令，运行在核心态
  • 应用程序：只能执行非特权指令

操作系统的内核

计算机上配置的底层软件，最重要、最核心、最基本的功能，时钟管理，中断管理，原语（驱动，cpu切换，最接近硬件的部分，程序运行具有原子性）

  • 时钟管理
  • 中断处理
  • 原语
  • 对系统资源进行管理的功能
    • 进程管理
    • 存储器管理
    • 设备管理

操作系统的体系结构

  • 大内核：将操作系统的主要模块都作为系统内核，运行在核心态
    • 优点：高性能
    • 缺点：内核代码大，结构混乱，难以维护
  • 微内核：只把最基本的功能保留在内核
    • 优点：内核功能少，结构清晰，方便维护
    • 缺点：需要频繁的在核心态和用户态之间切换性能低

3、中断和异常

概念和作用

中断是指计算机运行过程中，出现某些意外情况需主机干预时，机器能自动停止正在运行的程序并转入处理新的程序，处理完毕后又返回原被暂停的程序继续运行。

  • 当中断发生时，CPU会立即进入核心态
  • 当中断发生后，当前运行的进程暂停运行，并由操作系统内核对中断进行处理
  • 对于不同的中断信号，会进行不同的处理

发生了中断就意味着需要操作系统介入，开展管理工作，一般的管理工作（分配I/O设备、进程切换）需要使用特权指令，因此CPU需要切换到核心态，使操作系统获得计算机的控制权，有了中断才有了多道程序并发执行

  1. 用户态->核心态 是通过中断实现的，而且是唯一途径
  2. 核心态->用户态是通过一个特权指令修改程序状态状态字

分类

  • 内中断（也称为异常、例外、陷入，信号来源于CPU内部，与当前执行的指令有关）
    • 自愿中断：指令中断
    • 强迫中断：
      • 硬件故障
      • 软件中断
  • 外中断
    • 外设请求
    • 人工干预

外中断的处理过程

  1. 执行完指令后cpu会检查当前是否有外部中断信号
  2. 如果有，保存当前进程方便以后恢复
  3. 根据中断信号，转入相应的中断处理程序
  4. 恢复原进程的cpu环境并退出中断，返回原进程继续往下执行

4、系统调用

概念、作用

操作系统提供给应用程序（编程人员）使用的接口

作用：向操作系统发送请求，完成应用程序对资源相关的操作需求，保证系统的稳定性和安全性

系统调用和库函数区别

系统调用的过程

  1. 传递系统调用参数
  2. 执行陷入指令（用户态），执行陷入指令（只能在用户态执行）后会引发一个内中断，立刻转化为核心态
  3. 执行系统调用相应服务程序（核心态）
  4. 返回用户程序

总结

5、进程

定义

程序：就是一个指令序列

程序段、数据段、PCB三部分组成了进程实体（进程映像）。一般情况下，我们把进程实体称为进程。

注意：PCB是进程存在的唯一标志。

传统典型定义：

  1. 进程是程序的一次执行过程。
  2. 进程是一个程序机器数据在处理机上顺序执行时所发生的活动。
  3. 进程是具有独立功能的程序在数据集合上运行的过程，他是系统进行资源分配和调度的一个独立单位。

引入进程实体概念后：

可以把进程定义为：进程实体的运行过程，是系统进行资源分配和调度的一个独立单位。

组成

程序段：程序代码本身、运行时所需要的数据（全局变量，局部变量等）

PCB：进程的管理者（OS）所需要的数据，进程存在的唯一标志

  • 进程描述信息
    • 进程标识符PID：进程创建时分配的一个唯一的、不重复的ID，用于区分不同的进程
    • 用户标识符UID
  • 进程控制和管理信息
    • 进程当前状态
    • 进程优先级
  • 资源分配清单
    • 程序段指针
    • 数据段指针
    • 键鼠
  • 处理机相关的信息
    • 各种寄存器的值：进程切换时需要把进程当前的运行情况记录下来，例如程序执行到哪一句了

数据段（程序运行所需的数据）：存放程序运行过程中处理的各种数据

组织方式

  • 链接方式
    • 按照进程状态将PCB分为多个队列
    • OS持有指向各个队列的指针
  • 索引方式
    • 根据进程状态的不同，建立几张索引表
    • OS持有指向各个索引表的指针

特征

  • 动态性：最基本特征
  • 并发性
  • 独立性：资源分配、调度的独立单位
  • 异步性：各进程以不可预知的速度向前推进，可能导致运行结果的不确定性
  • 结构性

状态

三种基本状态：

  • 运行态（running）：占有CPU，并在CPU上运行
  • 就绪态（ready）：万事俱备，之欠CPU时间片的获取
  • 阻塞态（waiting/blacked）

其他状态：

  • 创建态（new）：进程在被创建，系统为其分配资源，初始化PCB
  • 终止态（terminated）：进程从系统中撤销，回收资源，撤销PCB

状态之间的转换

进程控制

基本概念

什么是进程控制？

实现状态之间的转换

如何实现进程控制？

  • 为了防止PCB信息和状态之间出现冲突，例如因为某种原因导致PCB没有被修改就被移动到目标状态的等待队列中而引发异常。

  • 使用 原语 进行进程控制，原语可以实现执行期间不被中断，这样的不被中断的操作叫“原子操作”

原语是怎么实现原子操作的？

  • 采用“关中断”和“开中断”指令实现。
  • 关中断指令执行后外部中断信号会被忽略。
  • 执行开中断指令后才可以被执行中断指令。

“开关中断”权限很大，是特权指令。

进程控制相关的原语

原语的工作：

  • 更新PCB信息
    • 所有的进程控制原语都一定会修改进程状态
    • 剥夺当前CPU使用权必然需要保存运行环境至PCB中，以便以后恢复运行
    • 某进程开始之前必然要恢复上一次中断时的运行环境
  • 将PCB插入合适的队列
  • 分配/回收资源

进程的创建

无 -> 创建态 -> 就绪态

  • 创建原语
    • 申请空白PCB
    • 为新进程分配资源
    • 初始化PCB
    • 将PCB插入就绪队列
  • 引起进程创建的事件
    • 用户登录：例如Linux这样的分时系统当有新用户登录
    • 作业调度：多道批处理系统中当有新作业放入内存中
    • 提供服务；用户向OS提出请求
    • 应用请求：用户进程主动请求

终止

各个状态 -> 终止态 -> 无

  • 撤销原语
    • 从PCB集合中找到终止进程的PCB
    • 若正在运行，立刻剥夺CPU，分配给其他进程
    • 终止其所有子进程
    • 归还该进程的资源至父进程或OS
    • 删除PCB
  • 引起进程终止的事件
    • 正常结束
    • 异常结束
    • 外界干预

阻塞和唤醒

阻塞原语和唤醒原语必须成对使用

阻塞：运行态 -> 阻塞态

唤醒：阻塞态 -> 就绪态

进程的阻塞

  • 阻塞原语
    • 找到待阻塞进程的PCB
    • 保护进程运行现场（保存在PCB的现场保护区），将PCB状态信息设置为“阻塞态”，暂时停止进程运行
    • 将PCB插入相应事件的等待队列
  • 引起阻塞的事件
    • 需要等待系统分配资源
    • 需要等待相互合作的其他进程完成工作

进程的唤醒

  • 唤醒原语
    • 在事件等待队列中找个PCB
    • 将PCB移出等待队列，设置进程状态为”就绪态“
    • 将PCB插入就绪队列，等待被调度
  • 引起进程唤醒的事件
    • 等待事件的发生：因为什么事件被阻塞，就被什么事件唤醒

切换

  • 切换原语
    • 将当前运行环境保存到PCB
    • 将PCB移入相应队列
    • 选择另一个进程执行，并更新其PCB
    • 根据PCB恢复新进程所需要的运行环境
  • 引起进程切换的事件
    • 当前进程时间片到
    • 有更高优先级的进程到达
    • 当前进程主动阻塞
    • 当前进程终止

进程通信

顾名思义就是进程之间的信息交换

我们知道进程是OS分配资源的单位（包括内存空间地址），因此各进程拥有的内存地址空间相互独立。

为了保证安全，各个进程之间不能相互访问地址空间，但是进程之间的信息交换又是必须要实现的，为了保证进程间的安全通信，操作系统提供了一些方法。

共享存储

操作系统为连个想要相互通信的进程开辟一个共享（互斥共享）空间，通过共享空间相互访问。

基于数据结构的共享

速度慢、限制多、是一种低级通信方式

基于存储区的共享

在内存中划出一块共享存储区，数据的形式、存放位置都是有进程控制，而不是由操作系统控制，相比之下这种方式速度更快，是一种高级通信方式。

消息传递

进程之间数据交换以格式化消息（message）为单位。通过操作系统提供的”发送/接收消息“两个原语进行数据交换。

message包括两个部分：

  • 消息头：
    • 发送进程ID
    • 接收进程ID
    • 消息长度
    • 消息类型等格式化信息（计网中发送的“报文”其实就是message）
  • 消息体：待交换的数据实体。

直接通信方式

每个进程都有一个消息缓冲队列。

发送进程直接将message打包好后，通过发送原语直接发送到接收进程的缓冲队列中。

间接通信方式（信箱通信方式）

操作系统维护一个“信箱”进程，这个信箱进程的缓冲队列中保存着来自各个发送进程发来的message，然后信箱进程通过message中消息头中的数据，将message发送到接收进程。

管道通信

用于连续读写进程的一个共享文件，又名pipe文件。其实就是在内存中开辟一个大小固定的缓存区。

  • 管道只能采用半双工通信（同一时刻只能单向传输）。
  • 各进程访问管道互斥。
  • 数据以字符流的形式写入管道，并且写入文件只有当管道写满了才会阻塞write()，读入文件只有当管道读空了才会阻塞read().
  • 未写满不允许读，未读空不允许写。
  • 数据一旦被读出就会从管道中抛弃，这就意味着读进程只能有一个，否则会有读到错数据的可能。

6、线程

定义

为什么要引入线程？

  • 因为传统的进程只能串行的执行一系列程序，而有的进程可能需要“同时”做很多事情。

可以把线程理解为轻量级的进程。线程是一个基本的cpu执行单元，也是程序执行流的最小单位，引入线程之后不仅进程之间可以并发，进程内部的各线程之间也可以并发，提升了系统的并发度。

引入线程后，进程只作为除CPU之外的系统资源分配单元。

引入后的变化

  • 资源分配、调度
    • 传统进程中，资源分配和调度的基本单位都是进程
    • 引入线程后，进程是资源分配的基本单位，线程是调度的基本单位
  • 并发性
    • 传统进程只能在进程之间并发
    • 引入线程后，线程之间也能并发，增加了系统的并发度
  • 系统开销
    • 传统的进程间并发需要切换运行环境，系统开销很大
    • 进程之内的线程切换不需要切换进程环境，系统开销小
    • 引入线程后并发带来的系统开销减小

重要属性

  • 线程是处理调度的单位
  • 多CPU计算机中，各个线程可占用不同的CPU
  • 每个线程都有一个线程ID，线程控制块（TCB）
  • 线程也有“就绪”“运行”“阻塞”三种状态
  • 线程几乎不拥有系统资源
  • 同一进程的不同线程之间共享进程的资源
  • 由于共享内存地址，进程内的线程之间通信甚至不需要系统干预

实现方式

  • 用户级线程：有应用程序和线程库实现。所有的进程管理工作都是有应用程序负责，在用户级线程中线程切换可以在用户态下完成，无需操作系统干预，在用户看来有多个线程，但是在操作系统看来没有线程（用户级线程对用户不透明对系统透明）
  • 内核级线程：线程的管理工作是由操作系统内核完成，线程调度切换都是操作系统管理，因此线程切换需要核心态下才能完成。内核级线程就是操作系统能看到的线程。
  • 同时支持用户级和内核级：可以

多线程模型

多对一模型

多个用户级线程映射到一个内核级线程

优点：

线程切换在用户空间即可完成，开销小，效率高

缺点：

当一个用户级线程被阻塞后，整个进程都会被阻塞，并发度不高。

一对一模型

优点：某一个用户级线程阻塞了，其他线程还能继续执行，并发能力强，多线程可在多核处理机上并行执行。

缺点：一个用户级进程会占用多个内核级线程，线程切换由操作系统内核完成，需要切换到核心态，线程管理成本高，开销大。

多对多模型

将n个用户线程映射到m个内核级线程上

集二者所长，即解决了并发度低的问题，又解决了线程管理开销大的问题。

7、处理机调度

调度：确定某种规则来决定处理任务的顺序

处理机调度：进程数量大于处理机个数，从就绪队列中按照一定的算法选择一个进程并将处理机分配给他运行，以实现线程的并发执行。

三个层次

高级调度（作业调度）

由于内存有限，有时无法将所有用户提交的作业放入内存，因此就需要确定某种规则来决定将作业调入内存的顺序。

按照一定的原则从外存上处于后备队列的作业中挑选一些作业放入内存中 ，给他们分配资源，并建立进程，使他们获得竞争处理机的权利。

高级调度是内存和外存之间的调度，每个作业只调入一次，调出一次，作业调入时候会建立相应的PCB，作业调出时会撤销PCB，高级调度主要是指调入问题，因为只有调入时机需要操作系统决定，调出时机一定作业完成时机。

中级调度（内存调度）

决定将哪个处于挂起状态的进程重新调入内存，一个进程可以被多次调出、调入内存，因此中高级调度发生的频率要比高级调度更高。

引入虚拟内存技术后，可将暂时不能运行的进程调至外存等待，等他重新具备了运行条件且内存又稍有空闲时，再重新调入内存。

这么做的目的是为了提高内存利用率和系统吞吐量。

暂时调到外存的内存等待状态为挂起状态，注意：PCB并不会一起调到外存，而是常驻内存。PCB中会记录进程在外存中存放的位置，进程状态信息，操作系统通过内存中的PCB来保持对各个进程的监控、管理。被挂起的进程PCB会放到挂起队列中。

低级调度（进程调度）

操作系统中最基本的调度，频率很高，一般几十毫秒一次，主要任务是按照某种方法和策略从就绪队列中选取一个进程，将处理机分配给他。

补充知识

进程的“挂起态”

暂时调到外存等待的进程状态（suspend）

挂起态又可细分为就绪挂起，阻塞挂起两种状态。

七状态模型

在原来五状态模型的基础上添加了两种挂起态

8、进程调度

时机

什么时候需要进程调度？

  • 当前运行的进程主动放弃处理结果
    • 正常终止
    • 发生异常终止
    • 主动请求阻塞（如等待I/O）
  • 当前运行的进程被动放弃处理机
    • 时间片用完
    • 有更紧急的事需要处理（如I/O中断）
    • 有更高优先级的进程进入就绪队列

什么时候不能进程调度？

  • 处理中断过程中
  • 进程在操作系统内核程序临界区中
  • 在原子操作过程中

临界资源：一个时间段内只允许一个进程使用的资源（互斥共享）。

临界区：访问临界资源的代码。

内核程序临界区： 一般是用来访问某种内核数据结构的，比如进程的就绪队列（由各种就绪进程的PCB组成）

内核程序临界区访问的临界资源如果不尽快释放的话，极有可能影响操作系统内核的其他管理工作，因此在访问内核程序临界区期间不能调度和切换。

而访问普通的临界区是可以的。

切换与过程

“狭义的进程调度”与“进程切换”的区别

狭义的调度：从就绪队列中选中一个要运行的进程。（这个进程可以使刚刚被暂停执行的进程，也可能是另一个进程，如果是后者就需要进程切换）

进程切换：一个进程让出处理机，让另一个进程占用处理接的过程。

广义的进程调度包含了选择一个进程和进程切换两个部分。

进程切换的过程需要做什么？

  • 对原来运行的进程各种数据保存
  • 对新的进程的各种数据恢复

注意：进程切换是有代价的，过于频繁的进行进程的调度，切换必然会使整个系统的效率降低，使系统大部分时间都花在进程切换上。

方式

非剥夺调度方式（非抢占式）

只允许进程主动放弃处理机，直到进程终止或主动要求进入阻塞态。

实现简单，系统开销小，但是无法处理紧急任务。

剥夺调度方式（抢占式）

当进程正在处理机上执行的时候，如果有一个更重要得或者更紧迫的进程需要使用处理机，即立刻暂停正在执行的进程，将处理机分配给更重要得进程。

适合于分时操作系统、实时操作系统。

9、调度算法评价指标

CPU利用率

利用率 = 忙碌时间 / 总时间

系统吞吐量

单位时间内完成作业的数量

系统吞吐量 = 总共完成了多少作业 / 总时间

周转时间

从作业提交给系统开始，到作业完成为止的这段时间间隔。

  • 作业在外存后备队列上等待作业调度（高级调度）的时间
  • 进程在就绪队列上等待进程调度（低级调度）的时间
  • 进程在CPU上执行的时间
  • 进程等待I/O操作完成的时间

周转时间、平均周转时间

（作业）周转时间 = 作业完成时间 - 作业提交时间

平均周转时间 = 个作业周转时间之和 / 作业数

带权周转时间、平均带权周转时间

带权周转时间 = 作业周转时间 / 作业时间运行时间 = 作业完成时间 - 作业提交时间 / 作业实际运行时间

平均带权周转时间 = 各作业带权周转时间之和 / 作业数

等待时间

指进程 / 作业 处于等待处理机状态时间之和，等待时间越长，用户满意度越低。

进程：进程建立以后等待被服务的时间之和，在等待I/O完成的期间其实进程也是在服务的，所以不计入等待时间

作业：在进程的基础上还要考虑作业在外存后备队列中等待的时间。

平均等待时间：加和 / 数量

响应时间

用户提交请求到首次产生响应所用的时间

10、调度算法

先来先服务FCFS

短作业优先SJF(shortest job first)

  • 分为抢占式和非抢占式
  • 抢占式SJF调度算法的平均等待时间和平均周转时间最少

高响应比优先HRRN

相应比：(等待时间 + 要求服务时间 ) / 要求服务时间

总结

  • 这几种算法主要关心对用户的公平性，平均周转时间，平均等待时间等评价系统整体性能的指标，但是是不关心“响应时间”的，也不区分紧急程度，因此对用户来说交互性很糟糕。
  • 因此这三种算法适用于早期的批处理系统

时间片轮转(RR，round-robin)

  • 如果时间片太大，使每个进程都能在一个时间片内完成，则时间片轮转调度算法退化为先来先服务算法，并且会增大进程响应时间。
  • 如果时间片太小会导致进程切换频繁

优先级调度算法

  • 就绪队列未必只有一个
  • 优先级可以选择是否动态调整，设置为动态优先级、静态优先级
  • 通常系统进程优先级高于用户进程
  • 前台进程优先级高于后台进程
  • 操作系统更偏好I/O型进程（I/O繁忙型进程），因为越重视io就越可以让io设备进入工作
  • 动态优先级根据什么调整呢？
    • 等待时间
    • 运行时间

多级反馈队列调度算法

  • 设置多级就绪队列，各级队列优先级从 高 到 低，时间片从 小 到 大。
  • 新进程到达时先进入第一级队列，按照FCFS的原则排队等待分配时间片。
  • 如果用完时间片，进程还未结束则将进程添加到下一级队列队尾。
  • 如果已经是最后一级，则重新放到最下级队尾。
  • 只有第 k 级队列为空时才能为第 k + 1 级队列队头进程分配时间片
  • 被抢占处理机的进程重新放回原队列队尾（当前进程时间片没有使用完就有新的高优先级进程进入，此时会优先运行高优先级队列进程）

总结

11、进程同步与互斥

同步：亦称为直接制约关系， 是指为了完成任务而建立的两个或者多个进程，这些进程因为需要在某些位置上协调他们的工作次序而产生的制约关系。

进程之间的直接制约关系就是源自于相互合作。

例如管道通信，一定是先写数据再读数据。

临界资源：一个时间段内只允许一个进程使用的资源（互斥共享）。

互斥：对于临界资源的访问必须是互斥的地进行。互斥也被称为间接制约关系。进程互斥值当一个进程访问临界资源时，另一个资源想要访问该临界资源的进程必须等待。当访问该临界资源的进程访问结束，释放该资源后另一个进程才能访问。

四个逻辑部分

  • 进入区
  • 临界区
  • 退出区
  • 剩余区

为了实现对临界资源的互斥访问，同时保证操作系统的整体性能，需要遵循一下原则：

  • 空闲让进
  • 忙则等待
  • 有限等待：对请求访问的进程，应保证能在有限时间内进入临界区（保证不会饥饿）
  • 让权等待：进程无法进入临界时立刻释放处理机
